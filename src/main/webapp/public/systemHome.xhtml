WELCOME TO CODENFORCE !

Release notes

  • When creating an NOV, the recipient choice dialog has direct links to the person and address search/add dialogs. Note that when you click these links the recipient choice dialog will disappear. But when you click the blue "choose addressee" button, your new links will be populated and ready for selection.
  • Case report list Now includes each citation's entire log history, so log your citations as they move through the system!
  • Session Address is now: gone! Instead, when you search for an address from the link in the main top bar, you'll be able to directly link your selected address to either your current property or current person right from the address searcher.
  • The unit manager on the property profile is now active, allowing for adding, editing, and removing property units. More importantly, once the unit list is complete, you can start a new occupancy period on that unit and start the inspection process.

Known issues

  • From SG: Default violation findings are not "sticking" for future violations
  • Occupancy Period start/end dates are not writing to DB correctly
  • Max upload size still around 5mb

WIP (Work in process)

  • Field inspections on code enforcement cases, field inspection violation migration from occupancy to code enf. case
  • Adding origination event to master case list on dashboard
  • Person link role selection (owner, tenant, etc.)

  • Platinum Citations are now live for final testing. Viewing an old citation or creating a new citation on CECase profile now allows you to enter a docket number, record individual violation statuses, and create a date-stamped log of each phase of the citation life cycle. Photos/documents can be uploaded directly. Person links coming soon.

  • File uploads on CE case are working again using our unified photo-doc interface. File size limit is still around 10mb, but the solution has been found for an upcoming release.

  • Events now can contain person links! After you create an event, then click view/edit in its display table, and then "Manage event-person connections "
  • Mailing address linking is now active. You are always linking either a property or a person to the session's current address, which is displayed under the session Property session box. Manage property address connections in the property profile page and the person address connections in the person profile dialog, accessed with "person - view/edit" in the Person session box.
