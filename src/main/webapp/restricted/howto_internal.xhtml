System users

Permissions systems overview

User page access is managed by the Glassfish servlet container security mechanism. Within the set of users with access to load page X, access to command buttons will vary based on user needs.

Privilege inheritance

In general, the authorized actions of a higher-level user (listed toward the bottom of the list) include all of those permitted to users at a lower authorization level. For example, client municipality staff can see everything a client municipality viewer can see.

Public

The most restricted, non-login based user. Not issued an authenticated user token by the server.

Authorized actions

  1. Submit new and retrieve existing code enforcement action requests
  2. Submit and retrieve previously submitted occupancy permit applications
  3. Attach note events to cases for which a public access control code has been issued
  4. View contact information for all participating municipality and umbrella organization offices

Client municipality viewer

Embodied in a town council person in an oversight but not managerial role in the system.

  1. View municipality-level statistics and query results
  2. Browse open and historic cases down to the case summary level
  3. Add communication notes to open cases

Client municipality staff

  1. View all municipality-specific data not restricted by authorized Officers
  2. Add communication and meeting events to both open and closed cases
  3. Authorize occupancy permit applications to proceed to inspection
  4. Run all municipality-level data queries

Data control duties

  1. Release case-level reports, permits, etc., to authorized people
  2. Designate communication events created by muni-staff users as publicly viewable
  3. Enable, disable, and re-key public access control codes

Code enforcement officer

The only non-developer user who can carry out enforcement actions in the system, which are enumerated below:

  1. Open code enforcement and inspection cases
  2. Attach violations to code enforcement cases
  3. Compose notices of violation
  4. Issue citations for violations
  5. Carry out inspection-related events

Data control duties

  1. Enable, disable, and re-key public access control codes
  2. Designate events as officer-only events
  3. Designate most event types as publicly viewable

COG Staff

Embodied in a person who works for an organization which serves several different municipalities and may field case-related inquiries.

Permissions

  1. Add message, meeting, and communication events to all cases in authorized municipalities
  2. Enable, disable, and re-key public access control codes
  3. Run multi-client municipality queries
  4. Change a case's manager
  5. Edit high-level code collections, such as code sources and code books

Data control duties

  1. Enable, disable, and re-key public access control codes
  2. Reset and audit municipality application review process settings

System Administrator

Designated to one user per service umbrella, which in 2018 and 2019 consists of a council of governments, such as the Turtle Creek Valley Council of Governments or the Steel Rivers Council of Governments.

  1. Carry out and audit all actions available to any system user
  2. Edit municipality-level settings and core data
  3. Configure system events of all shapes and sizes
  4. Create high-level code collections, such as code sources and code books
  5. Designated as the case manager for system-level data storage cases used to log data changes of various kinds
  6. View all system user action logs

Data control duties

  1. Monitor system data integrity ruthlessly

Developer

Granted system application-level access to both the sever and database to carry out all essential system life cycle events:

  1. Compiling source, pushing to server, configuring server
  2. Creating and restoring database backups
  3. Create users and grant any system permission available

Occupancy permits are created with this workflow

  1. A person with plain old internet access submits an occupancy permit application
  2. Payment entries are made whose totals equals or exceed the occupancy inspection fee
  3. A codeconnect user with occupancy verification permissions approves the application

A particular ordinance in any source does not contain any enforcement data, such as the standard compliance window length.

Code enforcement

Code enforcement action requests initiate the code enforcement case management process. An action request consists of the following information:

  • Municipality designation
  • Parcel/property identifier in that municipality
  • Description of the code enforcement concern, including a concern category
  • Contact information for the requesting person

Public vs. internal code enforcement action requests

Both non-logged in (i.e. public) users and internal users can submit a code enforcement action requests for processing by a code enforcement officer. The exact same form is used by both types of users. CodeConnect automatically enters the Contact information for users already logged into the system, thus streamlining the process and preserving data integrity for staff who may submit many action requests.

Public access to code enforcement request statuses

After submitting a code enforcement request, site users are provided with a six-digit number that called a Public Access Control Code. All significant case events are generally released by officers and thus viewable by people via their PACC, including events related to a code enforcement case that is opened in response to that request.
